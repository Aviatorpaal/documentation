Storage¶

Resources related to storage.

Volume¶

The Volume resource represents ZFS pools and UFS volumes.

List resource¶

GET /api/v1.0/storage/volume/¶

Returns a list of all interfaces.

Example request:

GET /api/v1.0/storage/volume/ HTTP/1.1
Content-Type: application/json

Example response:

HTTP/1.1 200 OK
Vary: Accept
Content-Type: application/json

[
  {
          "status": "HEALTHY",
          "vol_guid": "8443409799014097611",
          "used": "192.0 KiB (0%)",
          "name": "tank",
          "used_pct": "0%",
          "used_si": "192.0 KiB",
          "id": 1,
          "vol_encryptkey": "",
          "vol_name": "tank",
          "is_decrypted": true,
          "avail_si": "4.9 GiB",
          "mountpoint": "/mnt/tank",
          "vol_encrypt": 0,
          "children": [],
          "total_si": "4.9 GiB"
  }
]
Query Parameters:
 
  • offset – offset number. default is 0
  • limit – limit number. default is 20
Response Headers:
 
  • Content-Type – content type of the response
Status Codes:
  • 200 – no error

Create resource¶

POST /api/v1.0/storage/volume/¶

Creates a new volume and returns the new volume object.

Example request:

POST /api/v1.0/storage/volume/ HTTP/1.1
Content-Type: application/json

  {
          "volume_name": "tank",
          "layout": [
                  {
                          "vdevtype": "stripe",
                          "disks": ["ada1", "ada2"]
                  }
          ]
  }

Example response:

HTTP/1.1 201 Created
Vary: Accept
Content-Type: application/json

  {
          "status": "HEALTHY",
          "vol_guid": "8443409799014097611",
          "used": "192.0 KiB (0%)",
          "name": "tank",
          "used_pct": "0%",
          "used_si": "192.0 KiB",
          "id": 1,
          "vol_encryptkey": "",
          "vol_name": "tank",
          "is_decrypted": true,
          "avail_si": "4.9 GiB",
          "mountpoint": "/mnt/tank",
          "vol_encrypt": 0,
          "children": [],
          "total_si": "4.9 GiB"
  }
JSON Parameters:
 
  • volume_name (string) – name of the new volume
  • volume_add (string) – name of the volume to extend
  • encryption (boolean) – encrypted volume or not
  • encryption_inirand (boolean) – initialize disks with random data (slow)
  • layout (list) – list of vdevs composed of “vdevtype” (stripe, mirror, raidz, raidz2, raidz3) and disks (list of disk names)
Request Headers:
 
  • Content-Type – the request content type
Response Headers:
 
  • Content-Type – the response content type
Status Codes:
  • 201 – no error

Datasets¶

POST /api/v1.0/storage/volume/(int:id|string:name)/datasets/¶

Create dataset for volume id.

Example request:

POST /api/v1.0/storage/volume/tank/datasets/ HTTP/1.1
Content-Type: application/json

{
  "name": "myds"
}

Example response:

HTTP/1.1 202 Accepted
Vary: Accept
Content-Type: application/json

{
  "atime": "on",
  "avail": 3848491008,
  "comments": "Test",
  "compression": "lz4",
  "dedup": "off",
  "inherit_props": [
    "compression",
    "aclinherit"
  ],
  "mountpoint": "/mnt/tank/myds",
  "name": "tank/myds",
  "pool": "tank",
  "quota": 0,
  "readonly": "off",
  "recordsize": 131072,
  "refer": 90112,
  "refquota": 0,
  "refreservation": 0,
  "reservation": 0,
  "used": 90112
}
JSON Parameters:
 
  • name (string) – name of the dataset
  • comments (string) – user comments for the dataset
  • compression (string) – compression level (lz4, gzip-[1-9], zle, lzjb)
  • dedup (string) – dedup (on, off, inherit)
  • atime (string) – access time (on, off, inherit)
  • readonly (string) – read only (on, off, inherit)
  • recordsize (string) – recodsize (512, 1K, 2K, 4K, 8k, 16K, 32K, 64K, 128K, 256K, 512K, 1024K)
  • case_sensitivity (string) – files case sensitivity (on, off, inherit)
  • quota (string) – quota for this dataset and all children
  • refquota (string) – quota for this dataset
  • reservation (string) – reserved space for this dataset and all children
  • refreservation (string) – reserved space for this dataset
  • readonly – read only (on, off, inherit)
Response Headers:
 
  • Content-Type – content type of the response
Status Codes:
  • 201 – no error

PUT /api/v1.0/storage/permission/ HTTP/1.1

          "mp_path": "/mnt/tank",

          "mp_mode": "755",
          "mp_user": "root",
          "mp_group": "wheel",
